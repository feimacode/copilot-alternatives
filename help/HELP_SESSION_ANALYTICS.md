# Session Analytics

Browse your past chat sessions with full turn-by-turn detail — token counts, tool calls, timing, and more.

![Session dashboard overview](https://raw.githubusercontent.com/feimacode/copilot-alternatives/master/assets/screenshots/session-dashboard.png)

*The session dashboard turns historical chat activity into a searchable, explorable record of your work.*

---

## Sessions in the Sidebar

Under **Session Stats**, you’ll find your recent chat sessions. Each item shows:

- the date and a short session ID
- how many turns were exchanged
- the vendor and model that handled the session
- the estimated cost or credits consumed

The root node summarizes the combined value across all sessions, showing `cr` for Copilot activity and `$` for other vendors when both are present.

### Session Filters

Use the filter button next to **Session Stats** to narrow the list by:

- vendor
- model name
- date range

Run `Copilot Alternatives: Clear Session Filter` to reset the view.

---

## Session Detail Dashboard

Click any session in the sidebar to open the full detail view. It includes:

### Session Overview

- total tokens used (input + output)
- estimated cost, or AI credits if the session is from GitHub Copilot
- vendor and model information
- request count and timing details

### Turns Table

Each turn is shown in a table with columns such as:

| Column | What it shows |
|---|---|
| **#** | turn index |
| **Role** | user, assistant, or tool |
| **Input Tokens** | tokens sent to the model |
| **Output Tokens** | tokens generated by the model |
| **Total Tokens** | combined input + output |
| **Cost / Credits** | estimated cost or AI credits |
| **Latency** | full round-trip time |
| **TTFT** | time to first token |
| **Tools** | tool-call count |
| **Vote** | upvote/downvote status |

### Tool Calls and Prompt Breakdown

Turns with tool activity can be expanded to reveal which tools were called. The dashboard also includes a breakdown of prompt categories such as:

- system prompt instructions
- user query text
- attached files
- context files
- conversation history

---

## Tips

- use filters to find sessions from a specific model or time period
- review tool-call counts to understand how agentic a workflow was
- compare latency and TTFT to spot slower providers or models
- treat the dashboard as read-only analysis; it does not change your VS Code state
