# Session Analytics

Browse your past chat sessions with full turn-by-turn detail — token counts, tool calls, timing, and more.

---

## Sessions in the Sidebar

Under **Session Stats** in the sidebar, you'll find a list of your recent chat sessions. Each entry shows:

- **Date** and a short session ID
- **Turn count** — how many messages were exchanged
- **Vendor / Model** — which provider and model handled the session

### Session Filter

You can filter the session list to narrow down what you see:

- Click the filter icon next to "Session Stats" to open the filter dialog
- Filter by **vendor**, **model name**, or **date range**
- An active filter shows a filled filter icon in the sidebar
- Run `Copilot Alternatives: Clear Session Filter` to reset

---

## Session Detail Dashboard

Click any session in the sidebar to open its detail dashboard. The dashboard shows:

### Session Overview
- Total tokens used (input + output)
- Total estimated cost
- Session vendor and model
- Request count and timing

### Turns Table
A full table of every turn in the session:

| Column | Description |
|---|---|
| **#** | Turn index |
| **Role** | User, assistant, or tool |
| **Input Tokens** | Tokens sent to the model |
| **Output Tokens** | Tokens generated by the model |
| **Total Tokens** | Sum of input + output |
| **Cost** | Estimated cost for this turn |
| **Latency** | Time from request to full response |
| **TTFT** | Time to first token |
| **Tools** | Number of tool calls in this turn |
| **Vote** | Upvote/downvote status |

### Tool Calls
Turns that included tool calls show a details button. Click to see which tools were invoked.

### Prompt Category Breakdown
A pie chart visualizes how your input tokens were distributed across categories:
- **Modified system prompt** — Agent/extension instructions
- **User query** — Your actual chat messages
- **User attachment** — Files you attached to the conversation
- **Context files** — Workspace context added to the prompt
- **Conversation history** — Previous turns in the conversation

---

## Tips

- **Use session filters** to find conversations with specific models or from certain time periods
- **Check tool call counts** to understand how agentic your workflow is — sessions with many tool calls tend to consume more tokens
- **Review latency and TTFT** to compare performance across different providers and models
- **The session detail dashboard** is read-only and generated from stored data — it won't affect your VS Code state
